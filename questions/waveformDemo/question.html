Reference: D Flip-Flop

This non-editable reference diagram shows D flip-flop behavior. It uses signals-param="ref_signals"; the Q signal is precomputed and displayed read-only.

Part 1: D Flip-Flop — Cell Feedback (toggle input)

This is the same circuit as the reference above. It uses the default input-mode="toggle": click each cell to cycle between 0 and 1. Uses feedback="cell" (default) to show per-cell correct/incorrect overlays and a detail table with submitted and expected values.

Visual cue to notice: the Q waveform stays primary, while hover, corner badges, and light red/green grading accents stay secondary.

The dff_signals param uses the shorthand authoring API — values for read-only signals and initial + correct_answers for editable signals — instead of raw WaveDrom JSON.

Part 2: JK Flip-Flop — Row Feedback (toggle input)

A JK flip-flop behaves as follows on each positive clock edge:

JKQnext
00Q (hold)
010 (reset)
101 (set)
11Q' (toggle)

Uses feedback="row" to show per-signal row correctness with score badges ("All correct" or "X/Y correct"). Also demonstrates two editable signals (Q and Q') with toggle input.

Visual cue to notice: row mode avoids per-cell badge clutter and instead uses a softer row tint plus one summary pill at the right edge.

Part 3: D Flip-Flop with Enable — Table Feedback (toggle input)

A D flip-flop with enable only captures D when EN = 1. Otherwise Q holds.

Uses feedback="table" to show only an overall score summary (no per-cell or per-row detail). Also demonstrates multiple editable signals (Q and Q') with toggle input.

Visual cue to notice: the interactive waveform stays clean after grading because only the overall summary updates, not each individual row or cell.

Part 4: 2-Input AND Gate — Multiple Boxes per Clock Period (text input)

This is a combinational AND gate with output Y = A & B. Both inputs and the output use period="0.5", so they change twice per clock cycle.

Uses input-mode="text" with period="0.5": two answer boxes appear inside each full clock period, directly demonstrating sub-cycle text input with multiple answer boxes per clock period. See Part 5 for the same circuit using interactive toggle cells.

Visual cue to notice: this is the fallback mode for dense editable slots where text entry is still the clearest interaction.

Part 5: 2-Input AND Gate — Multiple Toggle Cells per Clock Period

Same AND gate as Part 4 (Y = A & B), this time using the default input-mode="toggle" with period="0.5". Click (or press Space) on each half-period cell to cycle between 0 and 1. Two toggle cells appear inside each full clock period, demonstrating sub-cycle interactive input.

Visual cue to notice: denser half-period rows add light separators to help track the smaller cells without overpowering the waveform trace.

Part 6: Tri-State Buffer — Unknown Outputs (text input)

A tri-state buffer drives Y = D when EN = 1. When EN = 0 the output is high-impedance, represented as x.

Uses input-mode="text" with allowed_values=["0","1","x"]: enter 0, 1, or x in each box. Text mode is used here so the typed value and WaveDrom-rendered value can be compared for a row where x (high impedance / unknown) is a valid answer.

Visual cue to notice: valid x answers should read differently from unanswered cells, so "unknown" and "not filled in yet" are not visually interchangeable.
